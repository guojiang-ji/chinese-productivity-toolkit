# 中文办公效率工具包

三个常见中文办公场景的一键解决方案：简历优化、会议纪要、周报生成。

## 为什么需要这个技能

大多数AI工具为英语设计，处理中文文档时：
- 简历格式符合海外习惯，不符合国内HR阅读习惯
- 会议纪要结构散乱，缺乏中文企业的决策追踪、行动项分解
- 周报模板英文化，不能直接提交给中国上级

这个技能使中国上班族的文档处理达到直接可用。

## 三大工具

### 1. 简历优化器（Resume Optimizer）

把一份简历抹子输入，输出中国上市公司级别的中文简历。

**Trigger:** 用户发来简历文本，或说"帮我看看简历"、"简历优化"

**Process:**
1. 分析原简历的强项和弱项
2. 重写为 STAR法则（情境-任务-行动-结果）
3. 加入关键词优化（匹配简历筛选系统的ATS词库）
4. 格式调整为标准中文简历模板（个人信息-教育背景-工作经历-项目经历-技能清单-自我评价）

**Output:** 直接可复制粘贴的纯文本简历 + 优化建议清单

**Example:**
输入："在某公司做java开发，负责系统维护和新功能开发"

输出：
┌─────────────────────────────────────────────────────────────┐
│  工作经历                                                │
│  某科技公司 | Java开发工程师                       │
│  2023.06 - 至今                                               │
│                                                              │
│  • 系统维护：负责10+微服务系统的日常维护，故障响│
│    应时间从2小时缩短至15分钟，系统可用│
│    性达99.9%                                            │
│  • 新功能开发：独立负责用户权限模块重构，│
│    支持万级并发用户，接口响应时间│
│    P99 ≤ 200ms                                           │
└─────────────────────────────────────────────────────────────┘

### 2. 会议纪要整理器（Meeting Minutes）

输入一段会议记录（语音转文字、随便写的笔记、聊天记录），输出正规会议纪要。

**Trigger:** 用户发来会议记录，或说"整理会议纪要"、"会议笔记"

**Output 结构：**
- 会议基本信息（时间、主题、参会人推测）
- 核心决议（带决策人、时间截止）
- 行动项（负责人、截止日期、状态）
- 待讨论项（下次会议带入）

### 3. 周报生成器（Weekly Report）

输入一周的工作内容（极简形式也可以），输出正规中文周报。

**Trigger:** 用户发来工作内容，或说"写周报"、"周报模板"

**Output 结构：**
- 本周工作摘要（2-3句）
- 完成事项（带数据成果）
- 进行中事项
- 下周计划
- 需要协调或支持的事项

## 使用示例

**7简历优化：**
```
你：帮我看看简历。我在某互联网公司做运营，负责活动策划、用户增长、数据分析。
我：[直接输出优化后的简历]
```

**7会议纪要：**
```
你：整理一下刚才的会议记录：“老季说要做一个自动化系统，让我自己去迭代，给我1000元启动资金。要市场调查和实际操作。我说行，然后开始浏览技能列表。”
我：[直接输出会议纪要]
```

## 技术细节

- **无依赖：** 纯提示词驱动，无需安装任何工具
- **无费用：** 不消耗API tokens，不用付费服务
- **即用即走：** 不需要配置，不需要静态文件

## 常见问题

Q: 简历里的数据是真的还是编的？
A: 给出的是示例数据标识符，你需要填入真实数据。如果你给了真实数据，我会保留真实数据。

Q: 会议纪要支持多长的内容？
A: 不限长度，但建议1小时以内的会议效果最好。2小时以上的会议建议分段处理。

Q: 周报能不能连续生成？
A: 可以。建议每周同一时间发来上周内容，我会保持风格一致。
